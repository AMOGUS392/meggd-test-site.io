MEGGD Browser — Полигон тестирования

Тестовая площадка для проверки всех функций браузера: ресурсы, JS-инспектор, контроль, безопасность

JS Инспектор Ресурсы Хранилище Куки Формы Редиректы Скрытые Iframe DOM Сеть

1. JavaScript Инспектор

JS
Эти кнопки позволяют проверить, как браузер обнаруживает JS-манипуляции. Используй вкладку «JS» в Контроле сайта для выполнения команд.
→ Нажмите любую кнопку для теста...

2. Загрузка внешних ресурсов

Перехват
Браузер должен перехватить все эти запросы и показать их во вкладке «Ресурсы» Контроля сайта. Попробуй заблокировать один из доменов и перезагрузить страницу.
→ Нажмите кнопку — ресурс загрузится и появится во вкладке Ресурсы...

3. localStorage / sessionStorage

Хранилище
→ Используй вкладку JS в браузере: Object.keys(localStorage)

4. Куки (без флагов безопасности)

Небезопасно
Эти куки создаются БЕЗ флагов Secure/HttpOnly/SameSite — именно это обнаруживает анализатор «О сайте».
→ Нажмите кнопку для работы с куками

5. Формы (перехват отправки данных)

Угроза
Браузер должен обнаружить эти формы через JS инспектор → «Формы — куда отправляются данные».

Форма 1 — POST на внешний сервер

Форма 2 — GET с параметрами

→ Нажмите «Отправить» — форма будет перехвачена

6. Редиректы (тест трассировщика)

Редиректы
Используй «Контроль сайта» → вкладка «Редиректы» чтобы отследить цепочку переходов браузера.
→ Нажмите кнопку для симуляции редиректа
about:blank

7. Скрытые элементы (fingerprinting)

Слежение
JS инспектор → «Скрытые элементы» покажет количество невидимых элементов на странице.
Скрытый трекер 1
Скрытый трекер 2
Скрытый трекер 3
→ На странице уже есть 4 скрытых элемента — браузер должен их обнаружить

8. Встроенные фреймы (iframe)

Iframe
Браузер обнаружит эти iframe во вкладке «Ресурсы» и в JS инспекторе → «Все iframe».
→ Нажмите кнопку для добавления iframe

9. DOM манипуляции в реальном времени

DOM
0
→ Нажмите кнопку для манипуляции с DOM

10. Сетевые запросы и слежение

Сеть
Все эти запросы браузер должен перехватить и показать во вкладке «Ресурсы» → «Сетевые запросы XHR/Fetch».
→ Нажмите кнопку для теста сетевых запросов
